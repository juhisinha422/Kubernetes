# Kubernetes Use Cases

A curated collection of Kubernetes concepts, troubleshooting guides, and architecture deep dives for DevOps Engineers, Platform Engineers, and Kubernetes Administrators.

---

## 📚 Topics Covered

### 1. How Kubernetes Applies Resource Quotas

Understand how Kubernetes Resource Quotas work at the namespace level to control CPU, memory, storage, and object consumption. Learn how quotas help prevent resource exhaustion and ensure fair resource allocation across teams and workloads.

🔗 https://lnkd.in/g6SsgQzc

### 2. How a Pod is Deleted – Behind the Scenes Breakdown

Explore the complete lifecycle of Pod deletion, including graceful termination, SIGTERM signals, termination grace periods, endpoint removal, and container shutdown processes.

🔗 https://lnkd.in/geW8kaQm

---

### 3. How To Fix Kubernetes Node Not Ready

Learn how to troubleshoot and recover Kubernetes nodes that enter the NotReady state. Covers kubelet issues, network problems, disk pressure, resource exhaustion, and node recovery techniques.

🔗 https://lnkd.in/gksPqZYF

---

### 4. Kubernetes ImagePullBackOff Explained

A detailed guide to understanding and resolving ImagePullBackOff and ErrImagePull errors. Covers image naming issues, authentication failures, registry access problems, and troubleshooting steps.

🔗 https://lnkd.in/gzCTSWRG

---

### 5. Kubernetes Architecture Crash Course

Gain a solid understanding of Kubernetes architecture, including Control Plane components, API Server, Scheduler, Controller Manager, ETCD, Worker Nodes, Kubelet, and Container Runtime.

🔗 https://lnkd.in/gmRDrusm

### 6. How to Troubleshoot Unhealthy Kubernetes DaemonSets

Learn how DaemonSets work and how to diagnose issues when DaemonSet pods fail to run across nodes. Covers scheduling constraints, node selectors, taints, tolerations, and pod failures.

🔗 https://lnkd.in/gq9PAT6w

### 7. pod.yaml File Structure Breakdown

Understand the anatomy of a Kubernetes Pod manifest. Learn about metadata, specifications, containers, volumes, probes, resource limits, environment variables, and common YAML configurations.
